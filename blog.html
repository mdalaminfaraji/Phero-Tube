1. Stateless Communication

REST APIs follow a stateless communication model, where each request from a client contains all the information needed, and the server does not store client state between requests.

2. Resource-Based

RESTful APIs are centered around resources, each identified by a unique URI. Resources can be any entity that can be identified and manipulated, such as data objects or services.

3. CRUD Operations

RESTful APIs support standard CRUD operations: Create, Read, Update, and Delete. These operations are performed using standard HTTP methods: GET, POST, PUT/PATCH, and DELETE.

4. Uniform Interface

RESTful APIs have a consistent and uniform interface, providing a standardized way to access and manipulate resources. This includes a common set of conventions for representing data, often in JSON or XML format.

5. HATEOAS (Hypermedia as the Engine of Application State)

HATEOAS is a constraint in REST architecture where clients interact with the application entirely through hypermedia provided dynamically by the application servers. This allows for self-discovery and navigation of the API.
